# 책임 연쇄 패턴 (Chain Of Responsibility Pattern)
## 책임 연쇄 패턴은 언제 사용해야 하나요 ?
요청을 보내는 쪽(Sender)과 요청을 처리하는 쪽(Receiver)의 분리하는 패턴 <br>

핸들러 체인을 사용해서 요청을 처리한다.

### 책임 연쇄 패턴의 장점과 단점은 ?
### 장점
- 원하는 형태로 구성하는게 가능하다.
- 개방 폐쇄의 원칙을 만족할 수 있다.
### 단점
- 디버깅이 어려울수 있다.

### 자바와 스프링에서 책임 연쇄 패턴이 사용된 경우
Java 에서는 서블릿 필터가 책임 연쇄 패턴으로 되어있다고 볼수 있다.
